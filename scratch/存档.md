

存档码在技术上无法区分大小写，但在原则上必须保持大小写。

存档系统输入和输出的东西，称为顶级存档码。

## 把顶级存档码解析为3级存档码：去除定位文本

在顶级存档码中，前9个字符是`/AnoMurSA`（SA代表第一版存档码）。结尾则是`/`。存档码必须从`/AnoMurSA`开始读取，直到下一个`/`为止。在`/AnoMurSA`到`/`之外可以包含一些无关字符，存档码只会从一个`/AnoMurSA`开始读取，直到下一个`/`为止。

这样，解析出来的东西是3级存档码。

例如：4级存档码`123/AnoMurSAhello world/456`转化为3级存档码，会得到`hello world`。

## 把3级存档码解析为2级存档码：反混淆

3级存档码是经过洗牌得到的混淆版本，需要反混淆才能变为2级存档码。3级存档码的开头9个字符是一个10进制数字，如`528641563`，这是一个随机种子，也就是混淆所用的秘钥。密钥必须是正整数。

通过一个简单的随机算法，将字符串洗牌，得到的就是密文；用同样的办法，把一个1到字符串长度的数字序列洗牌，得到的就是一个索引表，依照这个索引表，就能把密文反向洗牌。洗牌过后，得到的就是2级存档码。

例如，3级存档码`123456789ohlel`反混淆后可能是2级存档码`hello`。

## 把2级存档码解析为1级存档码：字典

2级存档码里包含一些特殊的字符，遇到这些字符时，需要将其展开为一个更长的序列。这是一种简单的压缩算法。

`&`对应`@_ksm_ano_dialog_`，`%`对应`@_ksm_ano_clue_`。

## 把1级存档码拆分为一系列数据段

1级存档码是一系列有序数据段组成的序列。每两个数据段之间都以分号`;`结尾，最后一个分号**不能省略**。

第一个数据段是 ksmd 编译时间戳，必须在编译时间戳相同的情况下，存档码才能安全地读取。

接下来总共应当有6个数据段，依次为：

* noted ko : @char | @clue
* dialog list : @dialog
* star clues : @clue
* trash clues : @clue
* unchecked clues : @clue
* chat current dialog : @dialog | null

由于目前存档中几乎所有数据都是`ksmd obj`（除了一个可空的），所以在数据段之间用`;`作分隔符应当是安全的。

总共有7个数据段。不能多也不能少。

## 把数据段解析为列表

列表数据段以`@`分割，因为正好我这儿的数据几乎都是`@ko`型。`["item1", "item2", "item3"]`对应`@item1@item2@item3`。

准确地说，`@`意味着一个列表项的开始。因此，开头有`@`而结尾不能有`@`。

chat current dialog 也必须以`@`开头。

按顺序把数据段拆进对应列表里，并且检查数据类型。读档完毕。
